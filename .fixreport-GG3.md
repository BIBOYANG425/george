## What changed

### 1. getTimeseries buckets by LA-local day (`src/admin/analytics.ts`)
- New exported pure helper `laDayKey(instant)` uses `Intl.DateTimeFormat('en-CA', { timeZone: 'America/Los_Angeles' })` (formatToParts) to get the LA wall-clock calendar day. DST-correct — the offset shifts an hour across PST/PDT, so a fixed -8 would misfile summer rows.
- getTimeseries now keys buckets by `laDayKey(created_at)` instead of `String(created_at).slice(0,10)` (UTC). Bucket labels walk back from today's LA date using UTC arithmetic on the pure calendar date (DST-free), so it always yields `days` sequential LA days with no gap/dup across a DST boundary.
- Test (`tests/admin/timeseries.test.ts`): `03:00Z` → previous LA day (PDT and PST cases); `07:30Z` summer stays the same LA day (the case a fixed -8 gets wrong); integration test with fake timers pins the 03:00Z row to `2026-07-06`, not the UTC `2026-07-07`.

### 2. Honest coverage (`analytics.ts` + `dashboard-html.ts`)
- getOverview `msgsWithTokens` numerator now `.eq('role','assistant').not('tokens_used','is',null)` — matches getSystemHealth. Denominator (`assistantTotal`) unchanged, so numerator and denominator count the same row population.
- Dashboard overview note now prints `messagesWithTokens / telemetry.assistantMessages` (was `/ totals.messages`), i.e. the SAME denominator as the `coveragePct` percentage, with wording updated to "条 assistant 回合带 token；分母与百分比一致".
- Test (`tests/admin/overview-coverage.test.ts`): asserts every token-bearing query also filters `role=assistant`, and that `messagesWithTokens/assistantMessages/coveragePct` = 40/50/80.

### 3. getDistributions windowing (`analytics.ts` + `dashboard-html.ts`)
- getDistributions adds `.order('created_at', { ascending: false })` before `.limit(10000)` → the most-recent 10k messages.
- UI labels: "工具使用分布" and "渠道分布" panel tags and the "对话用户" (users) panel tag all now carry "· 近 10k 条消息".
- Note on "retention": `getRetention` / `getOnboarding` are NOT wired to any API route or UI surface (router mounts only overview/timeseries/live/distributions/users/user/health/review/... ), so there is no rendered retention metric to label. Distributions + users are the two windowed surfaces that actually render.

### 4. Lightweight crisis-count endpoint (`src/admin/router.ts` + `dashboard-html.ts`)
- New `GET /admin/api/review/crisis-count` → `{ count }` computed ONLY from `getDistressQueue` (no flagged/fabrication/injection fan-out). When the radar is gated OFF (default) getDistressQueue short-circuits with no DB reads → `{ count: 0 }`.
- `/review` (full payload) is unchanged; `loadReview()` still calls it.
- Dashboard boot badge (`refreshCrisisBadge`) now calls `/review/crisis-count` and reads `d.count` instead of fanning out the full `/review`.
- Test (`tests/admin/crisis-count.test.ts`): 401 without token; `{ count: 0 }` with the radar off + a dummy Supabase client.

### 5. Auto-escaping template + esc single-quote (`dashboard-html.ts`) — durable XSS fix
- `esc()` now also escapes `'` → `&#39;` (keeps existing `& < > "`).
- Added a client-side auto-escaping `html\`\`` tagged template: every interpolation is `esc()`'d by default; `raw()` is the explicit opt-out for intentional HTML; nested `html\`\`` results and arrays compose without double-escaping; the branded result coerces to its HTML string via `toString` for `.innerHTML`.
- **Architecture note (important):** the client `<script>` lived inside a `String.raw\`…\`` template that interpolates `${}` at build time, so the original code deliberately avoided `${}` (pure string concatenation). To emit a real `${}`-based tagged template into the browser, I converted the outer wrapper from `String.raw\`…\`` to a plain template literal and doubled the only 9 pre-existing backslashes (3 lines: the `shortModel` regex and the two `onModelSel(\\'…\\')` handlers). Verified byte-identical output before/after that switch (0-byte diff). Full static-asset split stays deferred per the plan.
- **Builders converted to `html\`\``** (every builder interpolating user-derived data): `esc`+`html`+`raw` helpers, `barList`, `card`, `renderMsg`, `renderConvoMsg`, `renderObs`, `renderObsPanel`, `renderFlag`, `renderSuspect`, `renderDistress`, `renderInjection`, `renderCrisisPanel`, `ctrlBadges`, `modelOptions`, `controlsPanel`, `confirmDanger` (modal innerHTML), the `loadUsers` row map, and the `openUser` drawer body (student name/major, heartbeat fields, the 6 profile blocks, conversation).
- **Left as string-concatenation (listed per the plan's escape hatch):** the `load*` orchestration scaffolds (`loadOverview`, `loadLive`, `loadUsers` panel/table shell, `loadReview`, `loadSystem`, `openUser` outer try/catch) and `lineChart`. These interpolate only (a) outputs of the converted builders — now `raw`-branded objects that compose correctly via `toString`, (b) numeric/`fmt`/`ago`/`money`/`hm` values, (c) server-generated date strings, and (d) a few catch-block `e.message` values that remain explicitly `esc()`'d. None interpolate an un-escaped user-derived string.
