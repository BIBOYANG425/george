# Slice 0.5: Supabase Migrations Reconcile + RLS Audit

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reconcile the 35+ tables in the live Supabase project (`ujkaregrwrppaehvbahf`) with migration files across both `bia-admin/supabase/migrations/` and `george/supabase/migrations/`, establish a documented ownership split, and verify a fresh apply matches prod.

**Architecture:** The live Supabase project has tables that were created by three independent paths: (1) checked-in migrations in `bia-admin/supabase/migrations/`, (2) checked-in migrations in `george/supabase/migrations/`, and (3) direct CREATE TABLE statements applied via Supabase MCP / dashboard with no migration file. This plan dumps the live schema, categorizes every table by ownership domain (admin / agent / shared), writes the missing migration files in the correct repo, and adds a CI check that future drift is caught before merge. No table is dropped or altered in prod; this is additive bookkeeping.

**Tech Stack:** Supabase CLI 2.x (already installed in both repos), Postgres 15 (Supabase managed), bash + jq for the audit script, vitest for the drift test in george.

Create `~/Code/george/scripts/audit-migrations.sh`:
```bash
#!/usr/bin/env bash
# For each CREATE TABLE in data/prod-schema.sql, report whether the table is
# defined in george/supabase/migrations/ or bia-admin/supabase/migrations/.
# Output one row per table: <table>  <george|bia-admin|both|none>

set -euo pipefail

REPO_ROOT="$(git rev-parse --show-toplevel)"
DUMP="$REPO_ROOT/data/prod-schema.sql"
BIA_ADMIN_MIGRATIONS="$HOME/Documents/BIA 新生service/supabase/migrations"
GEORGE_MIGRATIONS="$REPO_ROOT/supabase/migrations"

if [[ ! -f "$DUMP" ]]; then
  echo "ERROR: $DUMP missing. Run scripts/dump-prod-schema.sh first." >&2
  exit 1
fi

# Extract bare table names from the dump.
PROD_TABLES=$(grep -E '^CREATE TABLE public\.' "$DUMP" \
  | sed -E 's/^CREATE TABLE public\.([a-zA-Z0-9_]+).*$/\1/' \
  | sort -u)

printf '%-40s %s\n' "TABLE" "DEFINED_IN"
printf '%-40s %s\n' "-----" "----------"

while IFS= read -r t; do
  in_george=0
  in_bia=0
  if grep -lqE "CREATE TABLE (IF NOT EXISTS )?(public\.)?$t\b" "$GEORGE_MIGRATIONS"/*.sql 2>/dev/null; then
    in_george=1
  fi
  if grep -lqE "CREATE TABLE (IF NOT EXISTS )?(public\.)?$t\b" "$BIA_ADMIN_MIGRATIONS"/*.sql 2>/dev/null; then
    in_bia=1
  fi
  if   [[ $in_george -eq 1 && $in_bia -eq 1 ]]; then loc="both (DRIFT)"
  elif [[ $in_george -eq 1 ]]; then loc="george"
  elif [[ $in_bia -eq 1 ]]; then loc="bia-admin"
  else loc="none (MISSING)"
  fi
  printf '%-40s %s\n' "$t" "$loc"
done <<< "$PROD_TABLES"
```

### Task 6: Write the drift test

**Files:**

- [ ] **Step 1: Write a vitest that compares the prod dump against the union of both repos' migrations**

Create `~/Code/george/tests/migrations.test.ts`:
```typescript
import { describe, it, expect } from 'vitest';
import { readFileSync, readdirSync } from 'node:fs';
import { join } from 'node:path';
import { homedir } from 'node:os';

const PROD_DUMP = join(process.cwd(), 'data/prod-schema.sql');
const GEORGE_MIGRATIONS = join(process.cwd(), 'supabase/migrations');
const BIA_ADMIN_MIGRATIONS = join(homedir(), 'Documents/BIA 新生service/supabase/migrations');

function tablesFromDump(): Set<string> {
  const text = readFileSync(PROD_DUMP, 'utf-8');
  const re = /^CREATE TABLE public\.([a-zA-Z0-9_]+)\b/gm;
  const out = new Set<string>();
  for (const m of text.matchAll(re)) out.add(m[1]);
  return out;
}

function tablesFromMigrations(dir: string): Set<string> {
  const out = new Set<string>();
  for (const file of readdirSync(dir)) {
    if (!file.endsWith('.sql')) continue;
    const text = readFileSync(join(dir, file), 'utf-8');
    const re = /CREATE TABLE (?:IF NOT EXISTS )?(?:public\.)?([a-zA-Z0-9_]+)\b/g;
    for (const m of text.matchAll(re)) out.add(m[1]);
  }
  return out;
}

describe('migrations reconcile (Slice 0.5)', () => {
  it('every prod table is defined in at least one repo\'s migrations', () => {
    const prod = tablesFromDump();
    const george = tablesFromMigrations(GEORGE_MIGRATIONS);
    const biaAdmin = tablesFromMigrations(BIA_ADMIN_MIGRATIONS);

    const missing: string[] = [];
    for (const t of prod) {
      if (!george.has(t) && !biaAdmin.has(t)) missing.push(t);
    }

    expect(missing).toEqual([]);
  });

  it('no prod table is defined in BOTH repos (no DRIFT)', () => {
    const prod = tablesFromDump();
    const george = tablesFromMigrations(GEORGE_MIGRATIONS);
    const biaAdmin = tablesFromMigrations(BIA_ADMIN_MIGRATIONS);

    const both: string[] = [];
    for (const t of prod) {
      if (george.has(t) && biaAdmin.has(t)) both.push(t);
    }

    expect(both).toEqual([]);
  });
});
```

## Notes for the engineer executing this plan

- **Why no per-table TDD here?** Migration reconcile is a "make the world match what's documented" operation, not a feature build. The drift test in Task 6 is the test. Per-table tests would be busywork.
- **Why two PRs across two repos?** The split-ownership architecture is the whole point. One PR per repo, both referenced from each other.
- **What if `supabase db dump` doesn't include everything?** Run with `--data-only=false --schema public --include-types=table,view,index,policy,function`. If a table doesn't appear in the dump, it might be in a non-public schema (`auth`, `storage`, `realtime` — these are Supabase-managed and should NOT be in your migrations).
